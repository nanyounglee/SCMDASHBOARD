## ① 이슈→협력사 귀속 — Airtable 변경 불필요 (실측 확인)

기존 명세서의 "수주처 lookup 필드 추가" 요청을 **철회**합니다. 실측 결과:

- issue.csv `movement_id` ↔ order.csv `movement_산출이동` 정확 조인: 2026년 이슈 344건 중 **297건(86.3%) 매칭, 충돌 0건** (한 movement는 정확히 한 협력사에 귀속)
- 기존 프로젝트 코드 접두어 추정 조인과 **판정이 다른 건이 142건(매칭 297건의 48%)** — 현재 협력사 품질 지표의 절반 가까이가 잘못된 협력사에 귀속되고 있었음
- 미매칭 47건(13.7%)은 전년 발주 movement 목록 추가 + 접두어 폴백으로 커버 예정

→ 대시보드 코드 수정만으로 해결 (Airtable·GSheet 모두 불필요). 단, **공급망 포트폴리오 Tier 점수의 이슈대응 산식은 월간 리포트와의 정합을 위해 기존 방식을 유지**하고, 협력사 상세·이슈율 매트릭스·재발 뷰 등 분석 화면부터 정확 조인으로 전환 — Tier 산식 변경은 리포트 담당과 기준 합의 후 진행.

## ②③ 원인분류 + CAPA — 대시보드 인라인 입력 권장

**3안 비교:**

| | ⓐ Airtable 필드 (원안) | ⓑ GSheet CAPA 대장 | ⓒ 대시보드 인라인 (권장) |
|---|---|---|---|
| 저장소 | 이슈 테이블에 필드 5개 | 별도 시트, 키=movement_id | `data/capa_overrides.json`, 키=movement_id |
| 입력 위치 | Airtable | 시트 | **이슈 목록 행에서 직접** (드롭다운·날짜) |
| 대시보드 유입 | 기존 CSV 파이프라인 | 아래 3경로 중 택1 | 즉시 (입력=반영, GitHub 자동커밋으로 팀 공유) |
| 갱신 지연 | 주 1회(일일화 필요) | 경로에 따라 상이 | **0** |
| 전제 조건 | 스키마 변경 권한 | 없음 | 없음 |

- ⓒ는 이미 검증된 패턴의 재사용입니다 — 고객인지이슈의 대응내용·실패비용 수기입력(`ci_overrides.json`), 진행현황 비고(`progress_notes.json`), 변경이력(`change_log.json`)과 완전히 동일한 구조 (localStorage 즉시 저장 + GitHub 토큰 보유자 열람 시 자동 커밋). `movement_id`가 안정적 키.
- CAPA 입력 주체가 SCM 담당자(=대시보드 사용자)라 입력 동선도 자연스럽습니다. 협력사 대책 회신은 현실적으로 메일·메신저로 받아 담당자가 기입하는 흐름 그대로.
- **기한 관리의 갱신 주기 문제도 소멸**: 원안(ⓐ)에서는 이슈 CSV 일일 갱신 워크플로가 필요했지만, 인라인 입력은 CAPA 데이터가 CSV를 거치지 않으므로 불필요. (신규 이슈 유입 자체는 기존 주 1회 + "지금 새로고침"으로 충분한지 운영하며 판단)

**GSheet(ⓑ)를 쓸 경우의 유입 경로 3가지** (참고):
1. 주 1회 수동 CSV export — 기한 관리 기능과 상충, 비권장
2. Apps Script → GitHub Contents API push — 이전에 "지금은 보류"로 결정한 GSheets 자동화의 재개가 전제
3. 시트 "웹에 게시" CSV URL을 대시보드가 직접 fetch — 실시간·구축 최소. **주의**: 시트가 링크 공개 상태가 됨 (이 저장소가 이미 공개 리포지토리라 발주·매입 CSV도 공개인 점을 감안하면 동등한 노출 수준이지만, 사내 규정 확인 권장)

## ④ FAI 승인 대장 — 인라인 기본, 사진 중심이면 GSheet

- 초도 후보 추출(신규 제품×협력사 조합 177개·신규 협력사 16개 자동 검출)은 어느 안에서든 대시보드가 담당.
- 승인 기록(상태/판정일/판정자/비고)만이라면 **대시보드 인라인**(`data/fai_log.json`)이 최소 구축.
- **한도견본 사진 관리가 핵심이라면**: GSheet 대장 + Google Drive 폴더(사진) + 링크 컬럼이 낫습니다. Airtable 첨부파일 필드가 원래 강점이었지만 GSheet+Drive로 동등하게 대체 가능. 디자인팀 등 대시보드 비사용자와의 협업도 시트가 유리.
- 유입은 위 ⓑ 경로 3택1 (승인 기록은 실시간성이 덜 중요해 주기 반영도 무방).

## ⑤ IQC 협력사 자가검사 — Google Forms → GSheet **(2026-07-20 프로브로 확정)**

**프로브 판정 완료** (`airtable-discover` §B, 2회 실행):
- 이슈_RAW 뷰가 걸린 테이블은 별도 이슈 테이블이 아니라 **movement(전체 입하/이동) 테이블 그 자체** — "movement 테이블 존재?"는 예
- 그러나 **최근 입하(2026.7.15~7.20) 표본 100건 중 이슈 아닌 94건에서 검수수량 기록 0건** — 검수 결과는 이슈 발생 시에만 기록됨. **전수 IQC 데이터는 Airtable에 존재하지 않음** → 뷰 추가 경로 폐기, 신규 수집 프로세스 필요

### 확정안: Google Forms 자가검사 성적서

협력사가 출하 전 자가검사 결과를 직접 제출 → 응답이 GSheet에 자동 적재 → 대시보드 유입.

**Form 문항 설계** (사진 문항 없음 — 파일 업로드를 넣으면 응답자 Google 로그인이 강제되어 협력사 부담. 불량 사진은 기존 채널 유지):

| # | 문항 | 유형 | 비고 |
|---|---|---|---|
| 1 | 협력사명 | **드롭다운** (필수) | 자유입력 금지 — sup.csv 표기와 완전 일치해야 대시보드 조인 가능. 옵션은 활성 협력사 목록으로 구성, 변동 시 갱신 |
| 2 | 발주번호 | 단답 (필수) | order.csv `발주번호`와 매칭되는 조인 키 — 협력사 발주서에 표기되는 번호 |
| 3 | 제품명 | 단답 (선택) | 참고용 |
| 4 | 출하수량 | 숫자 (필수) | |
| 5 | 검사수량(표본) | 숫자 (필수) | |
| 6 | 불량수량 | 숫자 (필수, 0 허용) | |
| 7 | 판정 | 객관식: 합격 / 조건부 출하 / 불합격 (필수) | |
| 8 | 검사일 | 날짜 (필수) | |
| 9 | 특이사항 | 장문 (선택) | 조건부 출하 사유 등 |

**GSheet → 대시보드 유입 (2택1)**:
- ⓐ **시트 "웹에 게시" CSV URL을 대시보드가 직접 fetch** — 실시간·수작업 0 (권장). 주의: 링크 공개 상태가 됨 (저장소가 이미 공개라 동등 노출 수준이나 사내 규정 확인)
- ⓑ 주 1회 수동 CSV export → `CSV/iqc.csv` — 기존 절차와 동일, 검수합격률은 월간 지표라 지연 허용 가능

**대시보드 반영 (Form·Sheet 생성 후 저장소 쪽 작업)**: `D.iqc` 로드 키 등록 → **검수합격률(전수판)** = 1−Σ불량÷Σ검사 (협력사별·월별), KPI 트래커의 기존 `quality_inspect`(이슈 건 편향 표본)를 전수판으로 교체, 협력사 상세·scorecard에 협력사별 합격률 병기, 제출率(발주 건 대비 성적서 제출 비율)도 함께 표기해 미제출 협력사 관리.
