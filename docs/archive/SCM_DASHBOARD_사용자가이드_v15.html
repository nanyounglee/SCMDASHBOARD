2. 기간선택 — 주별 · 월별 · 분기별 · 기간지정 v15 신규 통합

탭동작

분기별연도 드롭다운(2025년 데이터가 있으면 함께 표시) + 1~4분기 선택.
기간지정달력으로 시작일~종료일을 직접 지정합니다. "지난 주 / 지난 달 / 이번 주 / 이번 달" 빠른 선택 버튼도 제공됩니다.

매입금액은 항상 월 단위: 매입금액·매출 원본 데이터(세금계산서작성월)가 월 단위로만 존재해서, 주별·기간지정을 선택해도 매입금액 관련 수치는 선택 구간이 걸치는 달들을 기준으로 월 단위로 집계됩니다. 화면에 "(월 단위 집계)" 안내가 함께 표시됩니다.
13개 화면은 아직 월 단위만 지원: 발주 진행현황·원가 분석·제품별 발주수량 추이·협력사 현황·협력사유형별 제품군·품질 현황·하도급법 위험·견적계산기·AI 어시스턴트·리포트 생성·재고 현황·공급망 포트폴리오·KPI 목표·트래킹 화면은 동일한 선택기가 보이지만 주별/분기별/기간지정 탭에 마우스를 올리면 "월 단위만 지원 (추후 반영)" 툴팁이 뜨고 클릭되지 않습니다. 세부 필터 로직은 다음 버전에서 순차 반영될 예정이며, 그 전까지는 월별 탭만 사용하면 됩니다.
YoY 정확도: 전년(2025년) 비교는 현재 월 단위로 사전집계된 데이터를 사용해 근사치로 계산됩니다. 2025년 원본(행 단위) CSV가 확보되면 정확한 날짜 기준으로 자동 업그레이드될 예정입니다.

전체 / 품질 / 수량 / 운영 탭이슈 상세 + 대응 + 실패비용. v14 이전에는 "전체" 탭에서만 검색 결과가 유지되고 품질/수량/운영 탭으로 전환하면 검색이 풀렸는데, 이제 탭을 바꿔도 검색이 유지됩니다. v15.2 분기·다중 월·주간·기간지정 선택 시 선택 범위 전체가 정확히 표시됩니다(이전에는 마지막 달만 나오던 버그 수정).
v14 제품명 표시 개선: 이슈 목록에 표시되는 제품명이 issue.csv의 "입고물품" 컬럼을 우선 사용하도록 바뀌어 더 정확한 제품명이 보입니다(예: "톤앤톤쿨러백, 톤앤톤쿨러백, 지퍼, 실크, 실크" → "톤앤톤쿨러백").
v15.1 이슈 집계 기준일 변경: 품질이슈는 품질이슈내용업데이트, 운영이슈는 운영이슈내용 업데이트 날짜 기준으로 집계됩니다. 수량이슈는 기존과 동일하게 실제입하일 기준. 이로 인해 지난주에 입하되었더라도 이번 주에 이슈 내용이 업데이트된 건은 이번 주 집계에 포함됩니다. Airtable에서 내보낼 때 이 두 컬럼이 포함된 뷰를 사용하세요. 컬럼이 없는 구버전 CSV는 자동으로 실제입하일로 폴백됩니다.
v15.2 이슈현황 탭·KPI 집계 기준 통일: 이슈현황 탭(전체/품질/수량/운영)과 종합현황 KPI 카드가 이제 동일한 날짜 기준으로 집계됩니다. 1분기(1~3월)를 선택하면 KPI 카드와 탭 목록 모두 해당 3개월 이슈 전체를 보여줍니다.

7. 공급망 포트폴리오 v12 Tier 도입 · v13 평가기준 단일화 · 월 단위 고정(§2)

협력사를 QCD(품질·원가·납기 대응력) × 관계(협조성·대체가능성) 두 축으로 4분면 Tier를 매깁니다.

Tier기준의미
Tier1 · CoreQCD 높음 + 관계 좋음핵심 협력사 — 우선 유지·확대
Tier2 · PerformerQCD 높음 + 관계 보통성과는 좋으나 협조성 관리 필요
Tier3 · DevelopingQCD 낮음 + 관계 좋음관계는 좋으나 QCD 개선 지원 필요
Tier4 · GeneralQCD 낮음 + 관계 보통일반 관리 대상

기존 A/B/C/D 등급은 그대로 함께 표시됩니다. v13 평가 기준이 월간리포트 정식 기준(2026.07)으로 통일되어, 반기 누적 실적으로 계산하고 예외·대체 항목이 비어있으면 1점, 반기 발주가 0건이면 D등급으로 고정됩니다.

기능: "이번 달 발주실적 있는 협력사만 보기" 체크박스로 비활성 협력사를 걸러낼 수 있고, C/D등급 협력사는 화면 하단 "변경검토 리스트"에 사유와 함께 별도로 정리됩니다.

8. 협력사 현황 v12 상세모달 확장 · v13 개월수 표기 · v14 드릴다운 인라인 전환 · 월 단위 고정(§2)

협력사명을 클릭하면 목록이 인라인으로 확장되며 아래가 표시됩니다.

  • 연도별 매입추이 — 25년/26년 등 연도별 매입금액을 선 그래프로 비교
  • 이슈 상세 — 해당 협력사와 관련된 이슈를 최대 30건까지 표로 표시
  • v13 발주 거래 개월수 — 26년 기준 몇 개월 거래했는지(예: 7월 기준 7/7), 2개월 이상이면 "활성" 표기. 활성 카드 클릭 시 필터도 가능합니다.
  • v14 월별 매입금액/MoM 스트립 · 발주내역 테이블 — 협력사별로 월별 매입 추이와 MoM 증감을 한눈에 보고, 아래에서 발주내역 테이블을 월별로 필터해 볼 수 있습니다.

"가장 빈번한 발주 제품 Top 3" 드릴다운은 변경되지 않았습니다.

참고: 이슈 상세는 이슈 데이터에 협력사명이 직접 기록돼 있지 않아 프로젝트 코드로 추정 매칭한 결과입니다. 드물게 코드 표기가 다른 건은 상세에서 누락될 수 있습니다.

9. 매출 분석 — 제품별 발주수량 추이 v12

기존 "제품별 판매량 추이" 화면이 항상 비어 보였던 것은, 사용하지 않는 판매 데이터(sales_monthly)에 의존했던 설계 문제였습니다. 발주_RAW의 발주수량을 기준으로 계산해 매주 올리는 데이터만으로 화면이 항상 채워집니다.

10. 검색으로 데이터 찾기 v14 개선

상단 검색창에 협력사명이나 제품명을 입력하면 대시보드 전체(KPI·차트·표)가 실시간으로 필터됩니다.

  • 이제는 어디를 클릭해도 검색이 유지됩니다 — 발주 TASK 상세, 매입추이 월클릭 상세, 이슈 상세의 품질/수량/운영 탭 전환까지 모두 검색 결과 그대로 이어집니다.
  • 고객인지이슈도 협력사로 검색됩니다 — 고객인지이슈 데이터 자체에는 협력사 정보가 없지만, 프로젝트명·관련제품을 발주 데이터와 연결해 검색이 가능해졌습니다.

11. 현재 화면 CSV 다운로드 v14 신규

상단바의 다운로드(⬇) 버튼을 누르면, 지금 화면에 보이는 표를 검색·기간 필터가 반영된 그대로 CSV 파일로 받을 수 있습니다. 화면을 바꾸거나 검색어를 바꾼 뒤에 눌러도 항상 현재 보이는 내용 기준입니다.

12. CSV 저장 규칙

  • 구글스프레드시트/Airtable에서 쉼표로 구분된 값 CSV로 내려받습니다.
  • 파일명은 접두어 없이 고정 영문명(order.csv, parts.csv, goods_master.csv 등)으로 저장합니다.
  • 저장 위치는 저장소의 CSV/ 폴더입니다.
  • 구매전략파트 CSV 3종(재고 사전집계)은 항상 함께 갱신해야 합니다.
  • 코스트베이스(parts.csv)는 파츠 단가가 바뀔 때만 갱신하면 되며, 매주 갱신하지 않아도 다른 화면에는 영향이 없습니다.
  • v14 큰 CSV 파일은 채팅으로 붙여넣기보다 로컬 CSV/ 폴더에 직접 저장하는 것을 권장합니다 — 대용량 텍스트를 채팅으로 옮기면 일부 문자가 손상될 위험이 있습니다.
  • v14 원본 CSV의 컬럼(헤더)명이 추가/변경되면 docs/CHANGELOG_v{N}.md에 남겨서 팀에 공유합니다.

13. 문제 확인

상황확인할 것
DB원가비교 탭이 비어 있음parts.csv(코스트베이스)가 CSV/ 폴더에 있는지 확인합니다.
발주 진행현황이 전주 대비 비교가 안 됨전주치 progress_연도_W주차.csv가 CSV/에 남아있는지 확인합니다(archive_csv.ps1이 주 1회 아카이브로 이동시킴).
포트폴리오 Tier가 이상함분기별평가 CSV의 예외사항/대체가능 점수 컬럼이 최신인지 확인합니다.
협력사 이슈 상세에 일부 건이 안 보임프로젝트 코드 접두어 추정 매칭의 한계입니다 — 발주 코드와 이슈 코드 표기가 다른 경우입니다.
제품별 발주수량 추이가 비어 있음해당 기간 발주_RAW(order.csv)에 산출물(제품명)이 채워진 행이 있는지 확인합니다.
구매 재고 화면이 비어 있음재고 사전집계 CSV 3종이 CSV/ 폴더에 있는지, 파일명이 정확한지 확인합니다.
로컬 파일로 열면 자동 로드·월간 공지사항이 안 됨브라우저 보안 정책 때문에 file://에서는 CSV 자동 로드와 협력사 스냅샷(CSV_BANK/) 비교가 막힙니다. GitHub Pages URL에서 확인하거나 직접 업로드합니다.
고객인지이슈가 협력사 검색에 안 잡힘ci.csv의 프로젝트명이 order.csv의 project 값과 일치하는지, 관련제품 표기가 산출물명과 충분히 유사한지 확인합니다(§10).
