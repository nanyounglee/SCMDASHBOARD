펼쳐진 상세에 표시되는 내용:

  • 월별 필터 — 우측 상단 드롭다운에서 특정 월을 선택하면, 그 달의 매입액(전월 대비 증감률 포함)·발주내역·이슈 상세만 걸러서 볼 수 있습니다. "전체 기간"으로 되돌리면 전체 이력이 다시 보입니다.
  • 연도별 매입추이 — 25년/26년 등 연도별 매입금액을 선 그래프로 비교. 그래프 위에 월별 금액과 전월 대비(MoM) 증감률이 카드 형태로 나열됩니다(선택한 월은 파란 테두리로 강조).
  • 발주 제품 전체 — 실제 주문 이력이 있는 제품만 발주 횟수 내림차순으로 나열하고, 옆에 발주 수량도 함께 표시합니다(주문 이력이 없는 제품은 목록에서 제외됩니다).
  • 발주내역 — 개별 발주 건(발주번호·산출물·발주수량·공급가액·과업지시일자)을 최근순으로 나열. 월별 필터를 선택하면 그 달 건만 보입니다.
  • 이슈 상세 — 해당 협력사와 관련된 이슈를 최대 30건까지 표로 표시(월별 필터 적용 시 해당 월 건만).
  • 발주 거래 개월수 — 26년 기준 몇 개월 거래했는지(예: 7월 기준 7/7), 2개월 이상이면 "활성" 표기. 활성 카드 클릭 시 필터도 가능합니다.
참고: 매입 현황 탭 등 다른 화면에서 협력사명을 클릭했을 때 뜨는 상세 팝업은 그대로 유지됩니다 — 모달 → 드릴다운 전환은 "협력사 현황" 탭의 협력사 목록에만 적용됩니다.
참고: 이슈 상세는 이슈 데이터에 협력사명이 직접 기록돼 있지 않아 프로젝트 코드로 추정 매칭한 결과입니다. 드물게 코드 표기가 다른 건은 상세에서 누락될 수 있습니다.

9. 검색으로 데이터 찾기 v14 개선

상단 검색창에 협력사명이나 제품명을 입력하면 대시보드 전체(KPI·차트·표)가 실시간으로 필터됩니다.

  • 이제는 어디를 클릭해도 검색이 유지됩니다 — 발주 TASK 상세, 매입추이 월클릭 상세, 이슈 상세의 품질/수량/운영 탭 전환까지 모두 검색 결과 그대로 이어집니다.
  • 고객인지이슈도 협력사로 검색됩니다 — 고객인지이슈 데이터 자체에는 협력사 정보가 없지만, 프로젝트명·관련제품을 발주 데이터와 연결해 검색이 가능해졌습니다.

10. 현재 화면 CSV 다운로드 v14 신규

상단바의 다운로드(⬇) 버튼을 누르면, 지금 화면에 보이는 표를 검색·기간 필터가 반영된 그대로 CSV 파일로 받을 수 있습니다. 화면을 바꾸거나 검색어를 바꾼 뒤에 눌러도 항상 현재 보이는 내용 기준입니다.

11. CSV 저장 규칙

  • 구글스프레드시트/Airtable에서 쉼표로 구분된 값 CSV로 내려받습니다.
  • 파일명은 접두어 없이 고정 영문명(order.csv, parts.csv 등)으로 저장합니다.
  • 저장 위치는 저장소의 CSV/ 폴더입니다.
  • 구매전략파트 CSV 3종(재고 사전집계)은 항상 함께 갱신해야 합니다.
  • 코스트베이스(parts.csv)는 파츠 단가가 바뀔 때만 갱신하면 되며, 매주 갱신하지 않아도 다른 화면에는 영향이 없습니다.
  • v14 큰 CSV 파일은 채팅으로 붙여넣기보다 로컬 CSV/ 폴더에 직접 저장하는 것을 권장합니다 — 대용량 텍스트를 채팅으로 옮기면 일부 문자가 손상될 위험이 있습니다.
  • v14 원본 CSV의 컬럼(헤더)명이 추가/변경되면 docs/CHANGELOG_v{N}.md에 남겨서 팀에 공유합니다.
  • 신규 거래처(sup.csv)를 매달 새로 받아 교체할 때는, 새 파일을 덮어쓰기 전에 archive_csv.ps1을 먼저 실행하세요 — 교체 직전 상태를 CSV_BANK/sup_YYYY_MM.csv로 자동 스냅샷 저장해, 다음 달 월간 공지사항에서 신규/거래종료 협력사를 비교할 수 있게 해줍니다(§13).

12. 문제 확인

상황확인할 것
DB원가비교 탭이 비어 있음parts.csv(코스트베이스)가 CSV/ 폴더에 있는지 확인합니다.
발주 진행현황이 전주 대비 비교가 안 됨전주치 progress_연도_W주차.csv가 CSV/에 남아있는지 확인합니다(archive_csv.ps1이 주 1회 아카이브로 이동시킴).
포트폴리오 Tier가 이상함분기별평가 CSV의 예외사항/대체가능 점수 컬럼이 최신인지 확인합니다.
협력사 이슈 상세에 일부 건이 안 보임프로젝트 코드 접두어 추정 매칭의 한계입니다 — 발주 코드와 이슈 코드 표기가 다른 경우입니다.
제품별 발주수량 추이가 비어 있음해당 기간 발주_RAW(order.csv)에 산출물(제품명)이 채워진 행이 있는지 확인합니다.
구매 재고 화면이 비어 있음재고 사전집계 CSV 3종이 CSV/ 폴더에 있는지, 파일명이 정확한지 확인합니다.
로컬 파일로 열면 자동 로드가 안 됨브라우저 보안 정책 때문에 file:// 자동 로드가 막힐 수 있습니다. GitHub Pages URL에서 확인하거나 직접 업로드합니다.
고객인지이슈가 협력사 검색에 안 잡힘ci.csv의 프로젝트명이 order.csv의 project 값과 일치하는지, 관련제품 표기가 산출물명과 충분히 유사한지 확인합니다(§9).
월간 공지사항의 신규/거래종료 협력사가 항상 "비교할 이전 스냅샷 없음"으로 뜸CSV_BANK/에 지난달 sup_YYYY_MM.csv 스냅샷이 있는지 확인합니다. 매달 최소 1번은 새 명단을 반영하기 전에 archive_csv.ps1을 실행해야 스냅샷이 쌓입니다(§11).
