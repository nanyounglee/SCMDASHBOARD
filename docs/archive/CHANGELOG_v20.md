# SCM 대시보드 v20 변경 내역 (2026.07.24)

팀 공유용 — 변경 1건당 1줄 요약입니다. 상세 사용법은 `docs/SCM_DASHBOARD_사용자가이드_v20.html`, 구현 로직은 `docs/SCM_DASHBOARD_로직설명_v20.html` 참고.

v20은 **Airtable 자동 갱신 주기 변경 + 수동 새로고침 버튼 + 협력사·졸업/출시 변경 이력 시스템**을 담은 버전입니다.

## Airtable 자동 갱신 — 목요일 13:00으로 변경 + "지금 새로고침" 버튼
- 매주 자동 갱신 시각을 **월요일 09:00 → 목요일 13:00(KST)**로 변경했습니다
- 상단 데이터 업로드 바에 **🔄 지금 새로고침** 버튼이 생겼습니다 — 자동 갱신을 기다리지 않고 원하는 시점에 Airtable 원본을 바로 재수집할 수 있습니다. 클릭하면 GitHub Actions가 즉시 실행되고(1~3분), 완료되면 자동으로 최신 CSV를 다시 불러와 화면에 반영합니다
  - GitHub 연동 토큰에 기존 Contents 권한 외 **Actions 실행 권한**이 추가로 필요합니다(설정 방법은 GitHub 연동 설정 팝업에 안내)
  - 재고주간·매출월간·재고 KPI 3종(GSheets 소스)은 이 버튼의 대상이 아닙니다 — 이번 자동화는 Airtable 소스에만 적용됩니다

## 협력사·졸업/출시 변경 이력 시스템 (신규)
- 종합 현황의 월간 공지사항에서 **졸업 제품·출시 제품·신규 협력사·거래종료 협력사** 4개 항목이 더 이상 "상단에서 선택한 기간"이 아니라 **실제 변경일 기준으로 오늘부터 최근 30일**을 롤링으로 보여줍니다 — 다른 달을 조회 중이어도 최근 변경사항을 놓치지 않습니다 (품절 제품만 기존처럼 선택 기간 기준 유지)
- 감지된 모든 변경은 `data/change_log.json`에 **영구 기록**됩니다 — 카드 하단의 "전체 변경 이력 보기"를 펼치면 지금까지 누적된 전체 이력을 날짜순으로 확인할 수 있습니다
- 졸업/출시는 굿즈마스터(`goods_master.csv`)의 졸업일·출시일을 그대로 사용하고, 협력사 신규/거래종료는 **매주 자동 갱신 시 남는 협력사 명단 스냅샷**(`CSV_BANK/`)을 주 단위로 비교해 감지합니다 — 기존 "월 단위 대략 비교"보다 날짜 정밀도가 높아졌습니다
- **알아두실 점**: 협력사 스냅샷은 이번 버전부터 쌓이기 시작하므로, 협력사 신규/거래종료 자동 감지는 앞으로 몇 주간 자동 갱신이 누적되면서부터 정상적으로 작동합니다. 그 전까지는 기존처럼 수동입력(`MANUAL_VENDOR_TERMINATIONS`)으로 보강됩니다
