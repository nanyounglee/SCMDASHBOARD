1. 접속과 데이터 로드

브라우저에서 대시보드 URL에 접속하면 CSV/ 폴더의 고정 파일을 자동으로 읽어 KPI와 차트를 채웁니다. 임시 검증이 필요할 때만 상단 업로드 영역에 CSV를 직접 올리면 됩니다.

핵심: 구매 재고 화면은 원천 RAW인 inventory_weekly를 직접 계산하기보다, 구글스프레드시트에서 미리 집계한 대시보드용 CSV 3종을 우선 사용합니다. 이 3종이 없을 때만 원천 RAW를 브라우저가 직접 계산합니다(숫자가 약간 달라질 수 있음).

장기미회전/졸업 검토 후보

제외 범위 (유지): 매입검토와 시즌재고계획은 기존 구매 대시보드에서도 별도 테이블 기반의 확정 전 기능이므로 통합 범위에서 계속 제외됩니다. 화면은 남아 있으나 저장은 브라우저 임시저장(localStorage)뿐이며, 운영 원본은 여전히 구글스프레드시트입니다.

4. 공급망 포트폴리오 v9 변경

협력사를 TASK 건수·매입대금·이슈대응·예외사항·대체가능 5개 항목(각 3점, 총 15점)으로 평가해 A/B/C/D 등급과 Core/Strategic/Preferred/General 4분면으로 분류합니다.

v9부터: 등급 산정 기준이 반기(1~6월, 2Q까지) 데이터 기준으로 바뀌었습니다. 1분기·2분기 등급이 서로 다르면 두 분기 점수의 평균으로 등급을 다시 매깁니다. 연간 기준(TASK 400/100/11건, 매입 1억/5천만원)의 절반 수준으로 반기 기준값이 조정되어 있습니다.

C등급은 거래 축소·이관 검토 대상, D등급은 더 우선적으로 검토가 필요한 협력사로 표시됩니다.

5. CSV 저장 규칙

  • 구글스프레드시트에서 쉼표로 구분된 값 CSV로 내려받습니다.
  • 파일명은 접두어 없이 dashboard_period_summary.csv, dashboard_group_summary.csv, dashboard_sku_snapshot.csv로 저장합니다.
  • 저장 위치는 저장소의 CSV/ 폴더입니다.
  • CSV/_manifest.json에 등록된 고정 파일명이 자동 로드 기준입니다.
  • 구매전략파트 CSV 3종은 항상 함께 갱신해야 합니다 — 하나만 갱신하면 나머지 화면이 이전 데이터 또는 레거시 계산으로 어긋날 수 있습니다.
