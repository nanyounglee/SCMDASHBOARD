# SCM 대시보드 v21 변경 내역 (2026.07.20)

팀 공유용 — 변경 1건당 1줄 요약입니다. 상세 사용법은 `docs/SCM_DASHBOARD_사용자가이드_v21.html`, 구현 로직은 `docs/SCM_DASHBOARD_로직설명_v21.html` 참고.

v21은 **연도별 데이터 아카이빙 체계 + 협력사 YoY 비교 + 분기별평가 실데이터 반영**을 담은 버전입니다. 올해 데이터는 지금 로직 그대로 운영하고, 지난해 데이터는 아카이브에서 자동으로 불러와 비교지표로 쓰는 구조가 완성되었습니다 — 내년·후년에도 코드 수정 없이 그대로 굴러갑니다.

## 연도별 아카이빙 체계 (신규)
- 전년도 발주/이슈 원본이 `CSV_BANK/archive/{연도}/` 폴더에 고정 보존됩니다 (2025년분은 이번에 이동 완료)
- 대시보드는 매년 "실행 시점의 작년"을 자동 계산해 이 폴더를 불러오므로, **내년이 되면 자동으로 2026년 데이터가 YoY 기준이 됩니다** — 연도 하드코딩 제거
- **매년 1월 2일 새벽, GitHub Actions가 직전 연도의 order/issue CSV를 자동으로 아카이브**합니다 — 사람이 챙길 일 없음. 놓쳐도 수동 실행 가능하고, 이미 새해 데이터로 바뀐 파일을 잘못 보존하지 않도록 연도 검증이 들어 있습니다
- 같은 데이터가 세 군데(`order_2025.csv` / `SCM_발주_RAW(2025).csv` / `data_2025.json`)에 중복 관리되던 것을 아카이브 폴더 한 곳으로 통합했습니다

## 협력사 상세 — 전년 동기 비교 (신규)
- 협력사 상세 모달(협력사 현황 드릴다운 포함)에 **전년 동기 비교 표**가 생겼습니다: 발주건수·발주수량·매입금액·이슈건수·이슈율을 1월~당해 최신월 누적 기준으로 전년과 나란히 비교하고 YoY 증감을 표시합니다
- 표 아래에 **전년 연간 전체 실적**(발주·매입·이슈 총계)도 참고로 병기됩니다
- **연도별 매입추이 차트에 전년도 12개월 라인**이 겹쳐 그려져 계절성 비교가 가능해졌습니다
- 전년 발주 이력이 없는 협력사는 비교 블록이 표시되지 않습니다

## 공급망 포트폴리오 — 분기별평가 실데이터 + 클릭 인터랙션
- **분기별평가 CSV(`quarter_eval.csv`)가 자동 로드에 추가**되어 Tier 분류의 관계 점수(예외사항+대체가능여부)가 실데이터로 계산됩니다 — 기존에는 CSV 미탑재로 전 협력사가 기본값(2점)으로 고정되어 Core/Developing이 0이었음. 반영 후 Core 6·Performer 4로 월간 리포트와 일치 (Tier3/4 차이는 협력사 모수 범위 차이로 별도 논의)
- 상단 **Core/Performer/Developing/General 카드를 클릭하면** 아래 협력사별 상세 표가 해당 Tier로 필터링됩니다
- **4분면 매트릭스의 점을 클릭하면** 해당 협력사 상세 모달이 열립니다

## 이슈현황 — 탭 버그 수정 + 미입하율 추이표
- 품질이슈/수량이슈/운영이슈/제품×협력사 이슈율 **탭을 눌러도 빈 화면이 나오던 버그를 수정**했습니다 (탭 전환 시 패널 표시 클래스 누락 — 데이터는 정상이었음)
- **미입하율 추이표**가 추가되었습니다: 월별 전체TASK·미입하건수·미입하율에 전년동월·YoY 비교까지 표시 (2026 목표 1.56% 이하)
