# SCM 대시보드 v21 변경 내역 (2026.07.20)

팀 공유용 — 변경 1건당 1줄 요약입니다. 상세 사용법은 `docs/SCM_DASHBOARD_사용자가이드_v21.html`, 구현 로직은 `docs/SCM_DASHBOARD_로직설명_v21.html` 참고.

v21은 **연도별 데이터 아카이빙 체계 + 협력사 YoY 비교 + 분기별평가 실데이터 반영**을 담은 버전입니다. 올해 데이터는 지금 로직 그대로 운영하고, 지난해 데이터는 아카이브에서 자동으로 불러와 비교지표로 쓰는 구조가 완성되었습니다 — 내년·후년에도 코드 수정 없이 그대로 굴러갑니다.

## 연도별 아카이빙 체계 (신규)
- 전년도 발주/이슈 원본이 `CSV_BANK/archive/{연도}/` 폴더에 고정 보존됩니다 (2025년분은 이번에 이동 완료)
- 대시보드는 매년 "실행 시점의 작년"을 자동 계산해 이 폴더를 불러오므로, **내년이 되면 자동으로 2026년 데이터가 YoY 기준이 됩니다** — 연도 하드코딩 제거
- **매년 1월 2일 새벽, GitHub Actions가 직전 연도의 order/issue CSV를 자동으로 아카이브**합니다 — 사람이 챙길 일 없음. 놓쳐도 수동 실행 가능하고, 이미 새해 데이터로 바뀐 파일을 잘못 보존하지 않도록 연도 검증이 들어 있습니다
- 같은 데이터가 세 군데(`order_2025.csv` / `SCM_발주_RAW(2025).csv` / `data_2025.json`)에 중복 관리되던 것을 아카이브 폴더 한 곳으로 통합했습니다

## 협력사 상세 — 전년 동기 비교 (신규)
- 협력사 상세 모달(협력사 현황 드릴다운 포함)에 **전년 동기 비교 표**가 생겼습니다: 발주건수·발주수량·매입금액·이슈건수·이슈율을 1월~당해 최신월 누적 기준으로 전년과 나란히 비교하고 YoY 증감을 표시합니다
- 표 아래에 **전년 연간 전체 실적**(발주·매입·이슈 총계)도 참고로 병기됩니다
- **연도별 매입추이 차트에 전년도 12개월 라인**이 겹쳐 그려져 계절성 비교가 가능해졌습니다
- 전년 발주 이력이 없는 협력사는 비교 블록이 표시되지 않습니다

## 공급망 포트폴리오 — 분기별평가 실데이터 + 클릭 인터랙션
- **분기별평가 CSV(`quarter_eval.csv`)가 자동 로드에 추가**되어 Tier 분류의 관계 점수(예외사항+대체가능여부)가 실데이터로 계산됩니다 — 기존에는 CSV 미탑재로 전 협력사가 기본값(2점)으로 고정되어 Core/Developing이 0이었음. 반영 후 Core 6·Performer 4로 월간 리포트와 일치 (Tier3/4 차이는 협력사 모수 범위 차이로 별도 논의)
- 상단 **Core/Performer/Developing/General 카드를 클릭하면** 아래 협력사별 상세 표가 해당 Tier로 필터링됩니다
- **4분면 매트릭스의 점을 클릭하면** 해당 협력사 상세 모달이 열립니다

## 이슈현황 — 탭 버그 수정 + 미입하율 추이표
- 품질이슈/수량이슈/운영이슈/제품×협력사 이슈율 **탭을 눌러도 빈 화면이 나오던 버그를 수정**했습니다 (탭 전환 시 패널 표시 클래스 누락 — 데이터는 정상이었음)
- **미입하율 추이표**가 추가되었습니다: 월별 전체TASK·미입하건수·미입하율에 전년동월·YoY 비교까지 표시 (2026 목표 1.56% 이하)

## v21.1 (2026.07.20) — 품질관리 1차: 검수 유출률 + 재발 이슈 뷰
- **검수 유출률 (고객인지이슈 화면)**: 고객인지이슈마다 같은 프로젝트에 고객 등록 전 내부 이슈 기록이 있었는지로 **내부 선행검출/유출**을 구분 — KPI 3장(기간 건수·내부 선행검출·검수 유출률)과 목록의 검출 배지 추가. 내부 이슈 데이터가 없는 연도(2024 이전)의 고객인지이슈는 "판정불가"로 제외해 왜곡 방지. 프로젝트 코드 추정 조인 기반이므로 추세 관찰용
- **재발 이슈 뷰 (이슈현황 새 탭)**: 같은 제품×협력사 조합에서 이슈가 연도 불문 2회 이상이면 자동 추출(2025 아카이브+2026 통합) — 연도별 건수·유형 분포·최근 발생일, 양 연도 모두 발생 시 "연속 재발" 배지, 행 클릭 시 개별 이슈 펼침. 근본원인 분류 필드 도입 전까지 조합 반복을 재발 신호로 사용
- **CSV 폴더 정리**: 자동 갱신 도입으로 미참조가 된 한글 원본·중복 CSV 8종 삭제 (git 이력·CSV_BANK에 보존)
- **Airtable 필드 추가 명세서** 신규: `docs/AIRTABLE_품질필드_추가명세.md` — 이슈 테이블 수주처 lookup·근본원인분류·CAPA 필드 4종, FAI 신규 테이블, movement 검수 데이터 확인 절차. **Airtable 관리자 확인 필요**

## v21.2 (2026.07.20) — 여분(버퍼) 발주수량 추천 + 고객인지이슈 탭 통합
- **여분(버퍼) 발주수량 추천 (발주 현황 화면 신규)**: 제품(산출물)별로 당해+전년 이슈 이력에서 **수량부족률**(수량이슈내용 텍스트의 "N개 발주/M개 입하"·"N개 부족" 패턴 + 이동수량 컬럼 병행 집계)과 **품질불량률**(불량수량÷검수수량), **미입하율**을 산출해 **권장 버퍼%**(부족률+불량률 올림, 1~10%)와 권장 여분수량을 제안합니다
- 권장 버퍼를 같은 이력 기간에 적용했다고 가정한 **예방비용**(여분수량×단가×발주건수)과 실제 발생한 **실패비용**(재제작 취득원가 합계)의 **차액**을 제품별로 비교 — 차액이 양수(초록)면 버퍼 도입이 재제작보다 경제적. 단가는 파츠리스트 CSV 업로드 시 DB원가(BOM 합산) 우선, 없으면 평균매입단가(공급가액÷발주수량)
- **행 클릭 시 버퍼% 시나리오 비교**(1/2/3/5/10%+권장): 시나리오별 여분수량·예방비용·차액·판정을 한눈에 — "여분을 몇 % 잡을지" 의사결정용
- **고객인지이슈 탭 통합**: 별도 사이드바 화면이던 고객인지이슈가 **이슈 현황의 서브탭**(품질/수량/운영/고객인지이슈/제품×협력사/재발)으로 들어왔습니다 — 검수 유출률 KPI·연도별/작성자별 차트·대응/실패비용 수기입력 기능은 그대로
- **실패비용 현황 카드가 이슈 현황 본문으로 이동** — 어느 서브탭을 보고 있든 재제작+수기입력 실패비용 추이가 항상 보입니다
- **월간 공지사항 축소**: 기본 접힘(제목 옆 요약 한 줄: 🎓 졸업 · 🚀 출시 · 🚫 품절 · 🆕 신규 · 🔚 종료 건수) — 헤더 클릭으로 펼침/접힘, 선택은 브라우저에 저장. 펼침 상태도 박스·폰트를 컴팩트하게 축소
- **협력사·매입 현황 통합**: 별도 화면이던 **매입 현황**이 [공급망] 그룹 **"협력사 · 매입 현황"**의 서브탭(매입 현황/협력사 목록)으로 통합됐습니다 — 협력사별 Top10·제조유형/업태/인쇄 차트·매입상세 Top20, 협력사 목록·변경검토 진행현황 기능은 모두 그대로. [발주 · 매입] 그룹의 매입 현황 항목은 제거

## v21.3 (2026.07.20) — 미입하율 추이 차트 + 지점 클릭 MoM/YoY
- 이슈 현황의 미입하율 추이표 옆에 **추이 차트**가 추가됐습니다: 당해년(빨강) vs 전년(회색 점선) vs 목표선 1.56%(초록 점선)
