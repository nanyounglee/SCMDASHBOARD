# SCM 대시보드 v17 변경 내역 (2026.07.16)

팀 공유용 — 변경 1건당 1줄 요약입니다. 상세 사용법은 `docs/SCM_DASHBOARD_사용자가이드_v17.html`, 구현 로직은 `docs/SCM_DASHBOARD_로직설명_v17.html` 참고.

v17은 **협력사 현황 화면에 협력사 변경 검토 대시보드(scm_vendor_change)의 진행 현황을 실시간 연동**한 버전입니다.

## 협력사 변경 검토 진행 현황 카드 (신규)
- 협력사 현황 화면, 협력사 목록 위에 **협력사 변경 검토 진행 현황** 카드가 추가됩니다
- 별도 도구인 [협력사 변경 검토 대시보드](https://nanyounglee.github.io/scm_vendor_change/)에서 팀원들이 작성·저장한 협력사 전환 검토 보고서의 **전환 검토중 / 확정 / 완료** 단계별 건수와, **최근 완료된 협력사 변경 목록**(완료월·제품·변경 전 협력사·변경 후 협력사·확정일·완료일)을 이 화면에서 바로 확인할 수 있습니다
- 카드를 클릭하면(어디를 눌러도) 새 탭에서 협력사 변경 검토 대시보드가 열립니다 — 검토 배경·단가 비교·이슈 현황 등 상세 내용은 그 대시보드에서 확인
- **실시간 연동**: scm_vendor_change 저장소의 `bank/` 폴더(팀원들이 각자 컴퓨터에서 내보낸 검토 보고서를 공유하는 곳)를 GitHub API로 직접 읽어옵니다. 별도 CSV 업로드가 필요 없고, scm_vendor_change 쪽에서 새 보고서가 bank에 올라오면 이 카드도 자동으로 최신 상태를 반영합니다
- 같은 검토 건이 여러 팀원의 백업 파일에 서로 다른 진행 상태로 중복 등장하는 경우, 가장 진행된 상태(완료 > 확정 > 전환검토중)와 가장 최근 날짜를 채택해 병합합니다 — 백업 파일이 처리되는 순서에 따라 결과가 달라지지 않도록 검증 완료

## 운영·안정성
- 협력사 현황 화면에 처음 들어갈 때만 데이터를 불러오고(세션 내 캐시), GitHub API 무인증 호출 제한(시간당 60회)을 고려해 불필요한 재조회를 하지 않습니다
- 네트워크 오류·오프라인 상태에서는 "데이터를 불러오지 못했습니다" 안내로 우아하게 전환되며, 이때도 카드 클릭으로 대시보드 이동은 그대로 동작합니다
- 실제 bank 데이터(8개 백업 파일, 검토 40건·고유 10건)로 fetch·병합·렌더링을 검증 — scm_vendor_change 라이브 페이지의 표시값(전환검토중 3 / 확정 1 / 완료 6)과 정확히 일치함을 확인

## 다음부터
- 팀원이 scm_vendor_change에서 상태를 바꿨지만 아직 "내보내기 + bank 업로드"를 하지 않았다면, 그 변경은 이 카드에 반영되지 않습니다 — bank 업로드가 그 도구가 제공하는 유일한 교차-기기 공유 경로이기 때문입니다(그 도구 자체의 구조적 제약이며, 이번 연동으로 해결되는 범위 밖입니다)
- 대시보드를 변경하면 변경 내용을 다음 버전 `docs/CHANGELOG_v{N}.md`에 기록해 공유, 작업 시작 전 `git fetch && git log HEAD..origin/main`으로 원격 뒤처짐 확인
