v10→v12 변경 요약: 외주생산파트 화면 위주로 갱신됐다. ① 발주 현황 클릭 모달이 3-tab(발주현황/미발주현황/DB원가비교)으로 확장 ② 공급망 포트폴리오가 QCD×관계 기준 Tier로 재분류(활성 필터·변경검토리스트 추가) ③ 협력사 상세에 연도별 매입추이와 이슈 상세 추가 ④ 이슈 화면에 제품×협력사 이슈율 탭 신규 ⑤ 제품별 추이 화면이 "판매량"에서 "발주수량" 기준으로 변경. 구매전략파트(재고운영) 화면은 v10과 동일하다.

1. 접속과 데이터 로드

브라우저에서 대시보드 URL에 접속하면 CSV/ 폴더의 고정 파일을 자동으로 읽어 KPI와 차트를 채웁니다. 임시 검증이 필요할 때만 상단 업로드 영역에 CSV를 직접 올리면 됩니다.

핵심: 구매 재고 화면은 원천 RAW인 inventory_weekly를 직접 계산하기보다, 구글스프레드시트에서 미리 집계한 대시보드용 CSV 3종을 우선 사용합니다. 발주 상세분석의 DB원가비교 탭은 코스트베이스(파츠리스트) CSV가 있어야 채워집니다 — 없어도 다른 화면에는 영향이 없습니다.

2. 구매 재고 화면 (변경 없음)

재고운영 현황, 품절 상세, 장기미회전/졸업 검토는 v10과 동일하게 동작합니다. 매입검토·시즌재고계획은 계속 통합 범위에서 제외되며 화면만 유지됩니다.

3. 외주생산 화면 — 발주 상세분석 v11 변경

발주 현황에서 발주 TASK 숫자를 클릭하면 뜨는 모달이 3개 탭으로 나뉩니다.

탭보는 법
발주현황 Top전체 굿즈를 발주량 내림차순으로 확인(프로젝트/재고생산 구분) — 기존과 동일
미발주현황과거에 발주 이력이 있는 품목 중 이번 달 발주가 없는 품목을 모아서 보여줍니다. 신제품·비스포크는 대상에서 빠집니다.
DB원가비교발주수량 기준으로 코스트베이스(파츠 구성 단가)를 합산한 "DB원가"와 실제 매입금액을 비교해 절감율을 보여줍니다. 코스트베이스 CSV가 없으면 이 탭만 비어 있습니다.
DB원가란: 굿즈(제품) 하나를 구성하는 모든 파츠(본품/패키지/인쇄)의 표준 단가를 발주수량 구간에 맞춰 합산한 뒤 수량을 곱한 값입니다. 예를 들어 200개를 발주하면 100개 구간의 단가가 적용됩니다(발주수량 이하로 가장 큰 구간을 씀).

4. 이슈 관리 v12 변경

탭내용
전체 / 품질 / 수량 / 운영기존과 동일 — 이슈 상세 + 대응 + 실패비용
제품×협력사 이슈율 신규제품과 협력사 조합별로 "발주 건수 대비 이슈가 얼마나 자주 발생했는지"를 비율로 보여줍니다. 행을 클릭하면 해당 조합의 이슈 상세가 펼쳐집니다.

5. 공급망 포트폴리오 v12 변경 — 분류 기준 교체

기존에는 협력사를 "매입액 규모 × 등급"으로 4분면(Core/Strategic/Preferred/General)에 나눴습니다. v12부터는 QCD(품질·원가·납기 대응력) × 관계(협조성·대체가능성) 두 축으로 4분면 Tier를 매깁니다.

Tier기준의미
Tier1 · CoreQCD 높음 + 관계 좋음핵심 협력사 — 우선 유지·확대
Tier2 · PerformerQCD 높음 + 관계 보통성과는 좋으나 협조성 관리 필요
Tier3 · DevelopingQCD 낮음 + 관계 좋음관계는 좋으나 QCD 개선 지원 필요
Tier4 · GeneralQCD 낮음 + 관계 보통일반 관리 대상

기존 A/B/C/D 등급은 그대로 함께 표시됩니다 — Tier와 등급은 서로 다른 관점(협력 구조 vs 종합 성과)이라 병행해서 봅니다.

신규 기능: "이번 달 발주실적 있는 협력사만 보기" 체크박스로 비활성 협력사를 걸러낼 수 있고, C/D등급 협력사는 화면 하단 "변경검토 리스트"에 사유와 함께 별도로 정리됩니다.

6. 협력사 현황 — 상세 모달 v12 변경

협력사명을 클릭하면 뜨는 상세창에 아래가 추가됐습니다.

  • 연도별 매입추이 — 25년/26년 등 연도별 매입금액을 선 그래프로 비교
  • 이슈 상세 — 해당 협력사와 관련된 이슈를 최대 30건까지 표로 표시

"가장 빈번한 발주 제품 Top 3" 드릴다운은 기존 그대로입니다 — 협력사를 눌렀을 때 올해 누적으로 어떤 제품을 많이 발주했는지 보여주는 기능은 바뀌지 않았습니다.

참고: 이슈 상세는 이슈 데이터에 협력사명이 직접 기록돼 있지 않아 프로젝트 코드로 추정 매칭한 결과입니다. 드물게 코드 표기가 다른 건은 상세에서 누락될 수 있습니다.

7. 매출 분석 — 제품별 발주수량 추이 v12 변경

기존 "제품별 판매량 추이" 화면이 항상 비어 보였던 것은, 사용하지 않는 판매 데이터(sales_monthly)에 의존했던 설계 문제였습니다. v12부터는 발주_RAW의 발주수량을 기준으로 다시 계산하도록 바꿔, 매주 올리는 데이터만으로 화면이 항상 채워집니다. 화면 이름도 "발주수량 추이"로 바뀌었습니다.

8. CSV 저장 규칙

  • 구글스프레드시트/Airtable에서 쉼표로 구분된 값 CSV로 내려받습니다.
  • 파일명은 접두어 없이 고정 영문명(order.csv, parts.csv 등)으로 저장합니다.
  • 저장 위치는 저장소의 CSV/ 폴더입니다.
  • 구매전략파트 CSV 3종(재고 사전집계)은 항상 함께 갱신해야 합니다.
  • 코스트베이스(parts.csv)는 파츠 단가가 바뀔 때만 갱신하면 되며, 매주 갱신하지 않아도 다른 화면에는 영향이 없습니다.

9. 문제 확인

상황확인할 것
DB원가비교 탭이 비어 있음parts.csv(코스트베이스)가 CSV/ 폴더에 있는지 확인합니다.
포트폴리오 Tier가 이상함분기별평가 CSV의 예외사항/대체가능 점수 컬럼이 최신인지 확인합니다.
협력사 이슈 상세에 일부 건이 안 보임프로젝트 코드 접두어 추정 매칭의 한계입니다 — 발주 코드와 이슈 코드 표기가 다른 경우입니다.
제품별 발주수량 추이가 비어 있음해당 기간 발주_RAW(order.csv)에 산출물(제품명)이 채워진 행이 있는지 확인합니다.
구매 재고 화면이 비어 있음재고 사전집계 CSV 3종이 CSV/ 폴더에 있는지, 파일명이 정확한지 확인합니다.
로컬 파일로 열면 자동 로드가 안 됨브라우저 보안 정책 때문에 file:// 자동 로드가 막힐 수 있습니다. GitHub Pages URL에서 확인하거나 직접 업로드합니다.
