3. 핵심 파싱 로직

3-1. 발주 TASK 카운트 (KPI #1)

기준: 과업지시일자 월 기준 행 카운트 재고생산 판별: movement_산출이동에 '재고' 포함 또는 project에 '재고' 포함 프로젝트 발주 = 전체 - 재고생산 긴급 발주: 긴급여부 === '긴급' AND 수주처 !== '서울디지털인쇄협동조합'

3-2. 월 매입금액 (KPI #2) — 세금계산서 분할 처리

function purchaseByMonth(row): 1. 세금계산서작성월 (from 지출결의) 파싱 → 월 배열 2. 단일 월: 공급가액 전체 배정 3. 분할 월: 합계_분할 필드의 월별 금액 사용, 없으면 균등 분할

3-3. 입하경과일 자동 산출 (하도급법 연동)

function calcElapsedDays(row): 기준일 = 입하확정일 (from movement_산출물) || 입하예정일 (from movement_산출물) 종료일 = 실제입하일 (from movement) || 오늘 경과일 = (종료일 - 기준일) / 86400000 하도급법 위험 판별: 공급망_RAW의 하도급계약 대상여부 === '대상' AND 미입하 (실제입하일 비어있음) AND 경과일 >= 90일

3-4. 원가 자동 산출 (파츠마스터 + 공정DB · 원가분석 화면용)

1. sync_itemdb 필드에서 파츠코드 추출 (정규식: [A-Z]{4}_PT\d+) 2. 파츠코드 → parts_master.json에서 수량별 단가 조회 3. 공정DB → 굿즈코드별 공정 구성 + 표준단가 조회 4. 제품원가 = Σ(파츠별 구간단가 × 수량) 5. 마진율 = (판매가 - 원가) / 판매가

3-5. 실패비용 산출

function getReworkCostForProject(projName): 발주_RAW에서 project가 동일한 프로젝트의 행 중 task_id에 '재제작' 문자열이 포함된 행의 취득원가 합계 적용 위치: 이슈 현황, 고객인지이슈 모달, 이슈 모달, KPI 실패비용율(§3-16) v13 고객인지이슈는 위 자동 산출값에 수기입력 실패비용(ciExtraCost, CI_OVERRIDES)을 더해 합산한다(§3-19).

3-6. 전년 동기 비교 (YoY) / 3-7. 전월 대비 (MoM)

3-10. 협력사 평가 점수 산식 (원본, 5축 15점)

3-17. 고객인지이슈(ci) 검색 매칭 — 프로젝트 범위 산출물-수주처 조인 v14 신규

5. 검색 필터 연동

상단 검색창에 제품명/협력사명/파츠코드를 입력하면 모든 KPI와 차트가 실시간 필터됩니다(matchesSearch(), v14 출하장소 필드 추가로 17개 필드 대상 + 고객인지이슈 프로젝트범위 매칭(§3-17)).

v14에서 수정된 필터 누락 지점 정리 — KPI 카드는 검색을 반영하는데 그 카드를 클릭해 연 모달·서브탭·차트는 필터가 풀려 있던 지점들을 전수 점검해 수정했다:
  • 발주 TASK 상세 모달 (§3-11) — 기간 필터만 있고 검색어가 무시되던 문제
  • 매입추이 월클릭 상세 모달 (§4-1) — 클릭한 월 전체가 표시되던 문제
  • 이슈 상세 모달의 품질/수량/운영 서브탭 (§4-3) — "전체" 탭만 정상, 하위 탭 전환 시 필터가 풀리던 문제
  • 이슈 유형 분포 도넛, 월별 이슈 추이 차트 (§4-1) — 대시보드 화면 단계부터 필터가 전혀 적용되지 않던 문제
